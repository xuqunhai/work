Git第一次和远程建立连接时，出现下面错误：
permission denied (publickey)
原因是因为ssh key过期失效或者没有ssh key。
检查SSH key是否已经存在
ls ~/.ssh/
如果存在，就不用生成一个新的SSH key了，直接跳到下面的第3步。
不存在，生成一个新的SSH key
先
cd /Users/your_user_name/.ssh 
再
ssh-keygen -t rsa -b 2048 -C "your_email@example.com"
直接回车，会将key保存到默认文件中。
输入一个密码，以及确认密码，这个密码在你提交代码到Github时会用到
到这一步，你会发现 /Users/your_user_name/.ssh/id_rsa.pub 文件已经生成了。
将ssh key添加到git
把ssh 添加到keychain中，解决每次都需要输入密码
如果去git clone代码，会让你输入密码，如果一个还好说，如果关联了很多的话，那就比较麻烦了，这个时候的解决方法就是添加到keychain中：
ssh-add -K /Users/youre_user_name/.ssh/id_rsa

Mac 键盘符号说明
* ⌘ == Command 
* ⇧ == Shift 
* ⇪ == Caps Lock 
* ⌥ == Option 
* ⌃ == Control 
* ↩ == Return/Enter 
* ⌫ == Delete
* ⇞ == Page Up（Fn+↑）
* ⇟ == Page Down（Fn+↓）
* Home == Fn + ←
* End == Fn + →



格式调整
1. 上下移动一行： Alt+Up 或 Alt+Down
2. 在当前行下边插入一行 Ctrl+Enter
3. 在当前行上方插入一行 Ctrl+Shift+Enter

移动到单词最末尾：option + →
